# UAV-SDN Intelligent DDoS Detection System

> **Academic research project** for intelligent DDoS detection and mitigation in an UAV-SDN environment.

This project implements a complete experimental environment for detecting Distributed Denial of Service attacks in a UAV network using Software-Defined Networking, OpenFlow statistics, machine learning, and a real-time supervision dashboard.

---

## Overview

The objective of this project is to improve the **availability** and **resilience** of UAV communications against DDoS attacks.

The system monitors traffic generated by UAV nodes, extracts flow-level features from the SDN controller, applies a machine learning model to detect abnormal behavior, and reacts by applying mitigation rules through OpenFlow.

---

## Main Features

- UAV-SDN topology simulation using Mininet / Mininet-WiFi.
- Ryu SDN controller with OpenFlow 1.3.
- Real-time traffic monitoring.
- ML-based DDoS detection.
- OpenFlow mitigation rules.
- Web dashboard for supervision and control.
- Fleet monitoring with 50 UAV nodes.
- Link failure simulation.
- UAV battery and status monitoring.
- PPS normal / attack visualization.
- Trust score per drone.
- Graduated mitigation levels L0-L4.
- Explainable ML section.
- QoS monitoring.
- PCAP forensics support.
- FANET mobility simulation.
- Inter-swarm anomaly correlation.

## Components

### 1. SDN Controller

File:

```text
src/ryu_ml_ddos_v3.py
```

Responsibilities:

- Connects to Open vSwitch using OpenFlow 1.3.
- Collects traffic statistics.
- Extracts network features.
- Loads the trained ML model.
- Detects DDoS attacks.
- Applies mitigation rules.
- Sends real-time updates to the dashboard.

---

### 2. UAV-SDN Topology

File:

```text
src/topo_expert_uav.py
```

Responsibilities:

- Creates the UAV-SDN experimental topology.
- Supports 50 UAV nodes.
- Creates UAV swarms:
  - Reconnaissance swarm.
  - Surveillance swarm.
  - Logistics swarm.
- Adds GCS and Edge nodes.
- Provides REST APIs for topology control.
- Supports link status simulation and mobility.

---

### 3. Dashboard Server

File:

```text
src/dashboard_server_v3.py
```

Responsibilities:

- Hosts the dashboard interface.
- Exposes REST APIs.
- Manages WebSocket / Socket.IO updates.
- Synchronizes with Ryu and Mininet.
- Provides expert modules:
  - Trust score.
  - Mitigation levels.
  - QoS.
  - PCAP.
  - Mobility.
  - Correlation.

---

### 4. Web Dashboard

File:

```text
dashboard/dashboard.html
```

Main dashboard modules:

- Global system status.
- Fleet supervision.
- Drones and links table.
- Real-time PPS statistics.
- ML confidence history.
- Attack timeline.
- Expert intelligence panel.
- QoS and forensics panel.
- Light / dark mode.

## Connectivity Test

Inside the Mininet CLI:

```bash
recon1 ping -c 3 gcs
gcs ping -c 3 recon1
recon1 ping -c 3 surv2
```

Expected result:

```text
0% packet loss
```

Check OpenFlow rules:

```bash
sh ovs-ofctl -O OpenFlow13 dump-flows s1 | egrep 'NORMAL|drop|10.0.0.11'
```

## Launching an Attack

Example UDP flood from `recon1` to GCS:

```bash
recon1 python3 /tmp/attack_orchestrator.py \
  --type udp \
  --target 10.0.0.100 \
  --duration 20
```

Expected dashboard behavior:

```text
PPS increases
ML confidence increases
Drone status changes to ATTACKING
Trust score decreases
Mitigation level increases
OpenFlow DROP rule may be installed
```

## Expert Modules

### Trust Score

Each UAV receives a dynamic trust score:

```text
80-100  TRUSTED
55-79   SUSPECT
30-54   DANGER
20-29   CRITICAL
<20     BLOCK_RECOMMENDED
```

---

### Mitigation Levels

```text
L0  Normal
L1  Monitoring
L2  Rate-limit recommended
L3  Quarantine
L4  OpenFlow DROP
```
